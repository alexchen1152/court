Dave,

Yes, 3:30 works for me.

I have completed the original Shift4 Gateway interface to the new VeriFone MX platform/device driver and tested it with a test VeriFone MX device. I was able to run Sale, Auth, Return transactions.

I did not test Forced Sale, Adjust, Void, Settlements, reporting, and others.

The main issue with the porting of the terminal application to the new device is the old application was writtin in "C" language that will not run on the new device.

The VeriFone MX device uses an API that is based on the old XPI API used with terminal peripherals so that is good. For Shift4, this means that the source code to interface to the device is already done.

VeriFone also has a DLL module that can be used on the new device. We have built the simple test app and ran it on Friday on the VeriFone device. It works.

I think for CardConnect, we need to use the DLL for the new device.

The new device will include the following payment applications:

Standard Credit Debit Loyalty Gift Card EMV

Key injection is not required on VeriFone, the device can come pre= loaded with keys or keys can be remotely injected using existing protocols over the Shift4 network using 4Go.

We are also investigating a semi=integrated solution that would allow a PC to connected to the terminal, but the PC software is still in development.

Thanks,

JRock
