Prefetch

"119. Prefetch files are created by the operating system when an application is executed from a given location on a computer system for the first time. The purpose of a prefetch file is to speed up the loading of applications. Windows maintains a folder, typically found at C:\Windows\Prefetch, that contains the prefetch files."

"120. Prefetch files are a useful source of forensic data. A prefetch file contains the name of an executable, files and directories referenced by the executable, time the executable when executed, and the number of times the executable was run. It should be noted that, on a running system, the last execution times might be updated when the computer is shut down. Therefore, last execution timestamps might be slightly different on systems that were not shut down properly"

"121. The file name format for a prefetch file is typically: [executable name]-[hash value].pf"

Shellbags

"124. Shellbags track user view preferences of folders that a user has interacted with on a computer. The shellbag information can be found in the registry of a system in the USRCLASS.dat and NTUSER.DAT files."

"125. Shellbags track the first and last time a folder was interacted with by a user, as well as the number of times the folder was accessed by a user. Shellbag information shows artifacts of folders that no longer exist on a computer system. Therefore, shellbag information can show forensic artifacts of folders that were on USB storage devices connected to a computer. For example, if the contents of a USB storage device were copied onto a userâ€™s desktop and the userâ€™s desktop contains no folder with those files, then the shellbag information, which is stored in the registry, would contain artifacts of the history of the folder."

UserAssist

"128. UserAssist entries provide information about GUI applications that have been executed on a computer system. UserAssist entries are stored in the NTUSER.dat registry hive. Because the information is stored in the registry, even if the executable itself has been deleted from the system, the evidence that the program was run may still exist in the registry. Each time an executable is run, the operating system updates a registry key with the name of the excutable, the full path to the executable, and the last execution date and time. The information about UserAssist includes a count of the number of times the application was executed; however, this count does not always seem to be perfectly accurate. It is only incremented once per operating system boot cycle."
