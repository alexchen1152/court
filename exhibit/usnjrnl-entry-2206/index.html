Key Points and Observations:

  • USN Journal Entries: These entries provide metadata about file changes on an NTFS volume. The key pieces of information are:
    • File Name: The name of the file that was modified.
    • Time Stamp: The date and time (in UTC) of the modification.
    • Reason: USN_REASON_DATA_OVERWRITE indicates that data within the file was overwritten. This is typical when a file is edited and saved, or when a screenshot/image is captured and stored, replacing a previous version (if one existed at that exact location). Or the file was modified.
  • Text Messages: The recovered text message conversation discusses different ISVs (Independent Software Vendors) and their relationships with payment processors like First Data (FD), TSYS, and Chase. The conversation reveals details about how these ISVs manage their payment gateways, boarding, risk, and funding. The redactions likely protect the identities of specific companies.
  • The format of the files is PNG, and JPG.

Key Observations about Exhibit QQQ:

  • File Type: All files listed are .csv (Comma Separated Values) files, indicating tabular data.
  • File Naming Convention: The filenames suggest these are exports from a system called "Lighthouse Transaction Search" or "Shift4 Transaction ID export." The filenames also incorporate dates.
  • Update Reasons: The $UPDATE_REASON field provides a log of file modifications. Common reasons include:
    • DataOverwrite: Existing data in the file was overwritten.
    • DataExtend: New data was appended to the file.
    • BasicInfoChange: Metadata about the file (like its last modified time) was changed.
    • Close: The file was closed after being accessed/modified.
  • Chronological Tracking: The combination of $UPDATE_TIME and $SEQUENCE_NUMBER allows for precise tracking of when changes to each file occurred.

I have only included a small sample of the entire document. The full exhibit details hundreds of modifications to these CSV files over time. They show the changes occurring to the various data related files that Shift4 was using.
